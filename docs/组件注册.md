# 组件注册使用文档（`componentsRegistry` / `RegistryComponent`）

> 目标：在不改动原页面主逻辑的前提下，通过注册机制扩展页面 UI（按钮、Tab、内容块等）。

---

## 一、核心概念

### 1.1 注册中心

- 注册入口：`componentsRegistry.register(action)`
- 查询入口：`componentsRegistry.getRegistry(key)`
- key 规则：`${targetPage}:${targetModule || 'default'}`

例如：

- `targetPage: 'device/Instance/Detail'` + `targetModule: 'detail-tabs'`
- 对应 `RegistryComponent code="detail-tabs"`（页面路由名匹配 `targetPage`）

### 1.2 组件容器

页面通过 `RegistryComponent`（`RegisterComponents`）承载扩展点：

```vue
<RegistryComponent code="addTool" />
```

可选 `pageCode` 用于覆盖默认路由名匹配：

```vue
<RegistryComponent pageCode="layout" code="layout" />
```

---

## 二、注册数据结构

```ts
type ActionPosition = 'replace' | 'before' | 'after' | 'append' | 'hide'

interface RegistryAction {
  targetPage: string
  targetModule?: string
  target?: string
  mode?: ActionPosition
  order?: number
  component?: Component
  code: string
  props?: Record<string, any>
  extraOptions?: Record<string, any>
}
```

说明：

- `code`：注册项唯一标识；同一扩展点下重复 `code` 会被覆盖。
- `target`：目标 vnode 的 `key`；不传时可直接追加。
- `mode`：`replace / before / after / append / hide`，默认等效追加。
- `props`：传给注册组件的参数（也可放事件回调）。
- `extraOptions`：用于 `useRegistryOptions`（tabs/menu/list）的扩展项。

> 约定：以下所有 `component` 示例都使用异步组件 `defineAsyncComponent`。

```ts
import { defineAsyncComponent } from 'vue'
```

## 三、标准场景示例（每个场景一个示例）

### 场景 1：扩展 Tabs（追加）

父组件：

```ts
const baseTabs = ref([
  { label: '基本信息', value: 'Info' },
  { label: '设备数据', value: 'Running' }
])

const { mergedOptions } = useRegistryOptions({
  baseOptions: baseTabs,
  code: 'detail-tabs'
})
```

注册：

```ts
componentsRegistry.register({
  targetPage: 'device/Instance/Detail',
  targetModule: 'detail-tabs',
  code: 'alarm-tab',
  component: defineAsyncComponent(() => import('./tabs/AlarmTabView.vue')),
  extraOptions: { label: '告警信息', value: 'AlarmRecord' }
})
```

---

### 场景 2：替换指定 Tab（`replace`）

```ts
componentsRegistry.register({
  targetPage: 'device/Instance/Detail',
  targetModule: 'detail-tabs',
  code: 'replace-running',
  target: 'Running',
  mode: 'replace',
  component: defineAsyncComponent(() => import('./tabs/RealtimeTabView.vue')),
  extraOptions: { label: '实时数据', value: 'Running' }
})
```

---

### 场景 3：在目标组件前/后插入（`before` / `after`）

父组件：

```vue
<RegistryComponent code="addTool" is="a-space">
  <a-button key="add">新增</a-button>
  <a-button key="batch">批量操作</a-button>
</RegistryComponent>
```

注册（插到 `add` 后）：

```ts
componentsRegistry.register({
  targetPage: 'system/User',
  targetModule: 'addTool',
  code: 'custom-action',
  target: 'add',
  mode: 'after',
  component: defineAsyncComponent(() => import('./actions/CustomActionButton.vue'))
})
```

---

### 场景 4：不指定目标 `target` 直接追加

```ts
componentsRegistry.register({
  targetPage: 'layout',
  targetModule: 'layout',
  code: 'region-select',
  mode: 'append', // 可省略；默认会追加
  component: defineAsyncComponent(() => import('./components/RegionSelect.vue'))
})
```

这类注册会直接追加到 `RegistryComponent` 渲染列表末尾。

---

### 场景 5：`activeKey` 控制单个显示（保活）

```vue
<RegistryComponent code="detail-tabs" :activeKey="activeTab">
  <component :key="activeTab" :is="tabs[activeTab]" />
</RegistryComponent>
```

特点：

- 不卸载组件，只控制 `display`
- 切换后状态仍保留

---

### 场景 6：隐藏指定目标组件（`hide`）

父组件：

```vue
<RegistryComponent code="addTool" is="a-space">
  <a-button key="add">新增</a-button>
  <a-button key="batch">批量操作</a-button>
</RegistryComponent>
```

注册（隐藏 `add`）：

```ts
componentsRegistry.register({
  targetPage: 'system/User',
  targetModule: 'addTool',
  code: 'hide-add-action',
  target: 'add',
  mode: 'hide'
})
```

说明：

- `mode: 'hide'` 会直接移除命中的目标节点。
- 使用 `hide` 时可不传 `component`。
- 目标节点需要稳定 `key` 才能被准确匹配。

---

### 场景 7：使用 `is` 包装容器

```vue
<RegistryComponent code="addTool" is="a-space">
  <a-button key="add">新增</a-button>
</RegistryComponent>
```

`is` 可传字符串组件名或组件对象，统一包裹最终内容。

---

### 场景 8：注册组件触发父组件事件（事件透传）

父组件监听：

```vue
<RegistryComponent code="addTool" @save="reloadTable">
  <a-button key="add">新增</a-button>
</RegistryComponent>
```

注册（组件为异步）：

```ts
componentsRegistry.register({
  targetPage: 'system/User',
  targetModule: 'addTool',
  code: 'save-action',
  mode: 'append',
  component: defineAsyncComponent(() => import('./actions/SaveAction.vue'))
})
```

注册组件内部触发：

```vue
<script setup lang="ts">
const emit = defineEmits<{ (e: 'save'): void }>()
</script>

<template>
  <a-button type="primary" @click="emit('save')">插件保存</a-button>
</template>
```

说明：`RegistryComponent` 会透传 `onXxx` 监听器到注册组件，所以 `emit('save')` 可触发父组件 `@save`。

---

### 场景 9：通过 `props` 向注册组件传入回调

```ts
componentsRegistry.register({
  targetPage: 'system/User',
  targetModule: 'addTool',
  code: 'import-btn',
  component: defineAsyncComponent(() => import('./actions/ImportButton.vue')),
  props: {
    onSuccess: () => {
      console.log('导入成功后刷新列表')
    }
  }
})
```

---

### 场景 10：子模块统一注册入口（`getRegisterComponents`）

```ts
import { defineAsyncComponent } from 'vue'

const getRegisterComponents = () => [
  {
    targetPage: 'layout',
    targetModule: 'layout',
    code: 'region-select',
    mode: 'append',
    component: defineAsyncComponent(() => import('./components/RegionSelect/index.vue'))
  }
]

export default {
  getRegisterComponents
}
```

## 四、最佳实践

- 页面中被定位的基础节点尽量都加稳定 `key`。
- `registry` 只描述“扩展意图”，不要直接写业务状态逻辑。
- Tabs/List 扩展优先使用 `useRegistryOptions`；组件插槽扩展使用 `RegistryComponent`。
- 事件交互优先用 `emit + @event`；复杂场景再使用 `props` 回调。

## 五、常见问题

### Q1：注册了但没生效？

排查顺序：

1. `targetPage` 是否与当前页面路由名（或 `pageCode`）一致；
2. `targetModule` 是否与 `RegistryComponent code` 一致；
3. 使用 `target` 时，父组件对应节点是否有相同 `key`。

### Q2：为什么同一扩展点只保留了一条注册？

同一扩展点按 `code` 去重；`code` 相同会覆盖旧配置。

### Q3：`target` 不传可以吗？

可以。不传时会走追加逻辑，直接插入到末尾。
